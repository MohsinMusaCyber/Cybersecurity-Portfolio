# Dynamic Host Configuration Protocol (DHCP)

## Overview

Dynamic Host Configuration Protocol (DHCP) automatically assigns IP addresses and other network settings to devices on a network. This removes the need to configure each device manually.

---

## How DHCP Works

When a device joins a network, it requests an IP address from the DHCP server.

The DHCP process follows four steps:

1. **Discover** – The device broadcasts a request for an IP address.
2. **Offer** – The DHCP server offers an available IP address.
3. **Request** – The device requests the offered address.
4. **Acknowledge (ACK)** – The server confirms the assignment.

This is often remembered as **DORA**:

- Discover
- Offer
- Request
- Acknowledge

---

## Information Provided by DHCP

- IP Address
- Subnet Mask
- Default Gateway
- DNS Server
- Lease Time

---

## Why DHCP Is Important

- Automatically configures devices
- Reduces configuration errors
- Saves time for network administrators
- Simplifies network management

---

## Cybersecurity Relevance

Cybersecurity professionals monitor DHCP activity to identify rogue DHCP servers, unauthorised devices and suspicious network behaviour.
